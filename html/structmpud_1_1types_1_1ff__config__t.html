uint8_t mpud::types::ff_config_t::accel_on_delay

Specifies in milliseconds the additional power-on delay applied to accelerometer data path modules.

MAX = 3ms. More: The signal path contains filters which must be flushed on wake-up with new samples before the detection modules begin operations. There is already a default built-in 4ms delay.
